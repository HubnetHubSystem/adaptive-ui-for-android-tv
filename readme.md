# HubNet Adaptive UI for Android TV / Fire TV

## Concept
Phone apps often don’t work well on Android TV / Fire TV because they expect a touch screen. This project explores a **non‑intrusive adapter** that translates remote control input (DPad, OK, Back) into touch events, making any sideloaded phone app usable on TV without modifying the app itself.

## What's in this repo
- `detect_screen.py` – A simple Python script that demonstrates how developers can detect device type (phone, tablet, TV).
- `concept_hubnet_adapter.kts` – A Kotlin script that generates a two‑page HTML dashboard showing the idea in action.
- `config.json` – Example configuration (generated by the Kotlin script).
- `index.html` / `details.html` – Sample web pages that explain the adapter concept.

## How to use the Kotlin script (on your phone / Termux)
1. Install Kotlin: `pkg install kotlin`
2. Run the script: `kotlinc -script concept_hubnet_adapter.kts`
3. It will create `config.json`, `index.html`, and `details.html`.
4. Serve the files with Python: `python -m http.server 8082`
5. On your Fire TV browser, open `http://<your-phone-ip>:8082/index.html`

## Why this isn't a full solution (yet)
- Fire TV does not allow background accessibility services without sideloading.
- The adapter would need to be an Android app (compiled to APK) using `AccessibilityService` and `SYSTEM_ALERT_WINDOW`.
- This repo is a **theoretical blueprint** – not a working app.
